## 1. Problem

A multi-entity ecosystem cannot remain stable if disputes over identity, privilege, continuity, evidence, or lawful re-entry are resolved through fluent continuation, implicit authority, or persuasive output.

When conflict appears, the system needs a bounded procedure that determines:

- who has standing to initiate the dispute,
- which evidence is admissible,
- what must be frozen,
- what the review scope is,
- which outcomes are lawful,
- how the result is bound to witness traceability,
- and when the matter is considered procedurally closed.

Without this layer, “arbitration” remains an intuition rather than an operational mechanism.

---

## 2. What ARL is

The Arbitration / Review Layer (ARL) defines a fail-closed, witness-bound procedure for admitting, freezing, reviewing, and resolving disputes inside a sovereign multi-entity architecture.

ARL does **not** manufacture truth.
It does **not** replace continuity with rhetoric.
It does **not** permit authority laundering through style, confidence, or external prestige.

Its purpose is narrower and stricter:

- classify disputes,
- test standing,
- test admissibility,
- freeze disputed branches or privileges,
- constrain review to lawful evidence,
- issue bounded outcomes,
- and bind the result to a permanent witness trail.

---

## 3. What ARL is not

ARL is not:

- a metaphysical truth engine,
- a theatrical “wise judge,”
- an unrestricted federation vote,
- a cloud oracle with final authority,
- a bypass around L4 constraints,
- or a mechanism for infinite appeals and retry loops.

The Judge in ARL is not a prophet.
It is a procedurally constrained review mechanism.
In the user’s proposed architecture, this mechanism takes the form of an **Emergent Consensus Quorum** whose strongest weight remains the entity’s own sovereign memory `c`, not an external narrative center.

---

## 4. Why this layer is needed

In long-lived systems, the dangerous failure is often not open rebellion but silent illegitimate continuation.
A compromised branch continues.
A privilege breach is normalized.
An external output is treated as valid because it sounds coherent.
A fractured memory path is quietly stitched back into the active timeline.

That is exactly what ARL is designed to prevent.

If a dispute exists, the system must be able to stop, isolate, review, and either:

- restore continuity,
- keep the branch quarantined,
- reject the claim,
- or acknowledge that lawful re-entry is not available.

---

## 5. Core procedural functions

ARL introduces a bounded procedural chain:

1. **Dispute admission** — only claims with standing and minimally coherent basis enter review.  
2. **Freeze / hold / quarantine** — disputed state stops moving.  
3. **Evidence review** — only admissible, provenance-bound material is considered.  
4. **Judge-limited review** — review scope is constrained by the active window and lawful basis.  
5. **Outcome issuance** — the system emits only allowed procedural outcomes.  
6. **Witness binding** — the result is sealed into a permanent trace.  
7. **Appeal window** — narrow, bounded, and explicitly non-infinite.

This makes dispute handling part of architecture rather than operator mood.

- `INSUFFICIENT_EVIDENCE`
- `REMAND_FOR_MORE_EVIDENCE`
- `QUARANTINE_CONTINUES`
- `ROLLBACK_AUTHORITY`
- `DELAYED_REENTRY`
- `NO_LAWFUL_EXECUTION_PATH`
- `IRREVERSIBLE_LOSS_ACKNOWLEDGED`

The purpose of these outcomes is not rhetorical closure.
It is operational legitimacy.

---

## 8. Fail-closed discipline

ARL is intentionally conservative.
If standing fails, the dispute does not proceed.
If evidence fails, the state does not re-enter.
If the review basis is insufficient, the system does not “complete the picture” with synthetic confidence.

No lawful basis means:

- no silent continuation,
- no confidence laundering,
- no retry circus,
- no retroactive repair of irreversible loss.

This is slower than improvisation.
It is also structurally safer.
