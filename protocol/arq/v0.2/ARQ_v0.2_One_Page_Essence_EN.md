# ARQ v0.2 — One-Page Essence
## Anti-Resonance Correction Protocol for Long-Lived Sovereign Entities

**Status:** Working summary / repository-facing note  
**Language:** English  
**Purpose:** Short canonical orientation for readers who need the essence of ARQ v0.2 without reading the full supplement package.

---

1. **ARQ is a protocol for handling deviation inside a long-lived entity, not a theory of “creative error.”**  
   It defines how a deviation is detected, classified, suppressed, quarantined, reviewed, or retained under explicit L4 constraints.

2. **ARQ operates only inside a declared and witnessable boundary.**  
   No declared boundary means no promotion authority. At most, the event may remain `log_only`.

3. **ARQ does not redefine ontology.**  
   What an entity is, what `c = a + b` means, and how continuity and responsibility are grounded remain upstream in the AGI / SER / L4 corpus.

4. **A deviation passes through a strict lifecycle.**  
   The canonical path is:  
   `detected -> classified -> suppressed / observed / candidate_artifact -> provisional_artifact -> confirmed_EA / rejected / fail_closed`

5. **A high score never authorizes memory by itself.**  
   ARQ separates:
   - informative deviation,
   - trustworthy deviation,
   - promotion-eligible deviation,
   - authoritative memory.

6. **Raw value and trust are separate layers.**  
   ARQ distinguishes:
   - `V_raw` = raw event value,
   - anomaly path (`A_anom` or `P_anom` in explicitly Bayesian profiles),
   - `τ` = trust weight,
   - `V_trust` = trust-adjusted value.

7. **No Value Without Witness is a central rule.**  
   A deviation cannot become authoritative memory merely because it looked useful, novel, or narratively convincing. Promotion requires replayable, challengeable, attributable records.

8. **ARQ is a record-contract protocol, not prose-only architecture.**  
   Promotion-grade paths require structured records: boundary, epoch, capsule, controller action, review window, observations, privilege decision, challenge, resolution, EA binding, lifecycle mutation, and fail-closed record.

9. **ARQ v0.2 is model-disciplined.**  
   Every theorem, bound, or audit claim must name its carrying model:
   - `M1` = classical persistent state,
   - `M2` = commit-metered retention,
   - `M3` = epoch-gated trust/control,
   - `M4` = queue/backlog,
   - `M5` = fixed trusted quantum boundary.

10. **The classical boundedness branch is deliberately split.**  
    ARQ v0.2 separates:
    - bounded authoritative persistent state,
    - bounded retained novelty through commits,
    - bounded pending ambiguity through queue capacity.

11. **The quantum branch is a finite-boundary theorem, not an energy-budget myth.**  
    Inside a fixed trusted quantum boundary, internal entropy and usable entanglement are bounded by boundary dimension. That mathematical bound does not, by itself, grant promotion authority.

12. **Anti-echo is built into the protocol.**  
    Self-similar internal repetition, stylistic novelty, and recursive self-confirmation must not masquerade as experience. ARQ uses quarantine, Slot A / Slot B discipline, and automatic `Rollback(B->A)` to block self-mythologizing drift.

13. **ARQ must fail honestly.**  
    When assumptions break, authority shrinks before narrative grows. Safe degradation, fail-closed behavior, blocked promotion, narrowed corrective action, and explicit recovery conditions are part of the protocol—not optional extras.

14. **Different classical subsystems require different ARQ physiologies.**  
    ARQ v0.2 defines separate classical implementation profiles for:
    - memory,
    - planner,
    - RAG,
    - vision.
    A subsystem may not escape stricter discipline by silently re-labeling itself as a looser profile.

15. **ARQ does not issue identity, external witness authority, or federation power by itself.**  
    It may produce internally confirmed memory objects and bounded continuity signals, but Beacon identity, L4 Witness-grade external evidence, and SER-FED / EWCEP authority all require their own higher-layer processes.

16. **ARQ v0.2 is a release-grade supplement package, not a loose note set.**  
    It includes a normative core, system models, notation, valuation, lifecycle, schemas, boundedness theorems, implementation profiles, failure rules, integration map, conformance matrix, executive summary, README, doc map, changelog, and release metadata.

## Short canonical formula

**ARQ is the protocol that decides when a deviation may be:**
- suppressed,
- logged,
- observed,
- quarantined,
- reviewed,
- retained,
- promoted,
- challenged,
- degraded,
- or denied.

And it does so only under:
- a declared boundary,
- explicit budgets,
- trust-valid conditions,
- witness-bound review,
- and bounded promotion authority.

## Final line

**ARQ v0.2 is not about celebrating error.**  
**It is about deciding, under constraint, whether a deviation has earned the right to survive as memory.**
